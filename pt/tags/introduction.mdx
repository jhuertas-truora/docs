## Resumo

A marcação de perfis é um componente fundamental do CDP que permite a classificação e segmentação de clientes através da atribuição de tags específicas. Essas tags atuam como marcadores que representam atributos-chave, comportamentos ou características de um perfil de cliente, facilitando a criação de audiências dinâmicas e a execução de ações personalizadas.

## Princípios Chave:

- **Dinâmico e Atualizável**: As tags são dinâmicas e podem mudar ao longo do ciclo de vida do cliente, refletindo seu estágio atual, interesses e comportamento. A API deve permitir a adição e remoção ágil de tags.
- **Acionável e Informativo**: O objetivo principal da marcação é fornecer informações úteis que possam ser aproveitadas tanto para análise de dados quanto para ativação de campanhas e personalização em tempo real.

## Quais Casos de Uso as Tags Habilitam?
- Segmentação de usuários para campanhas personalizadas.
- Criação de regras que ativam ações automatizadas baseadas em uma tag.
- Análise de audiências para descobrir padrões e comportamentos.
- Melhoria da efetividade da tomada de decisões baseada em dados.

## Tipos de Tags

### Tags de Comportamento
Essas tags são atribuídas baseadas nas ações que um usuário realizou em sua plataforma. Exemplos incluem:
- `active-user`: Usuários que interagiram recentemente
- `high-engagement`: Usuários com alta participação
- `churn-risk`: Usuários que mostram sinais de abandono

### Tags de Segmentação Demográfica
Essas tags categorizam usuários baseadas em características demográficas:
- `premium-customer`: Clientes com planos premium
- `enterprise`: Clientes empresariais
- `trial-user`: Usuários em período de teste

### Tags de Estado do Produto
Essas tags refletem o estado atual do usuário em relação ao seu produto:
- `onboarding-complete`: Usuários que completaram o processo de integração
- `feature-adopter`: Usuários que adotaram novas funcionalidades
- `support-required`: Usuários que precisam de assistência

## Melhores Práticas para Marcação

### Convenções de Nomenclatura
- **Consistência**: Use um formato consistente para todas as tags (ex., kebab-case: `high-value-customer`)
- **Clareza**: Os nomes devem ser autoexplicativos e fáceis de entender
- **Escalabilidade**: Considere como as tags crescerão com seu negócio

### Gestão de Tags
- **Auditoria Regular**: Revise periodicamente as tags para eliminar as obsoletas
- **Documentação**: Mantenha documentação clara sobre o propósito de cada tag
- **Controle de Acesso**: Defina quem pode criar, modificar e excluir tags

## Integração com Outros Componentes

As tags se integram estreitamente com outros componentes do CDP:

- **Segmentação**: As tags são a base para criar segmentos dinâmicos
- **Personalização**: São utilizadas para personalizar experiências do usuário
- **Automação**: Podem ativar fluxos de trabalho automatizados
- **Analytics**: Fornecem dimensões para análise e relatórios
