## Resumen

Una Plataforma de Datos de Clientes (CDP) es un sistema de software que recopila, unifica y organiza los datos de clientes de múltiples fuentes de productos de Truora en un perfil único, consistente y completo para cada cliente. A diferencia de los almacenes de datos tradicionales o CRMs, una CDP está diseñada para ser accesible tanto para equipos técnicos como no técnicos, permitiendo la segmentación en tiempo real, personalización y activación de datos a través de canales.

## Descripción de la API

La API de CDP proporciona acceso seguro y centralizado a los datos de clientes almacenados en tu Plataforma de Datos de Clientes. Te permite:
- Recuperar y gestionar perfiles de clientes, incluyendo atributos personales y variables de decisión.
- Acceder y analizar transacciones asociadas con cada perfil.
- Etiquetar y categorizar perfiles con etiquetas personalizadas para mejorar la segmentación.
- Crear y gestionar segmentos dinámicos de clientes para acciones dirigidas e insights.

Esta API está diseñada para ayudarte a unificar la información del cliente, optimizar la toma de decisiones basada en datos y activar insights en todo el ecosistema de tu organización.

- Los beneficios clave incluyen:
- Acceso centralizado a datos para una vista completa del cliente 360°.
- Segmentación y etiquetado en tiempo real para gestión flexible de audiencias.
- Endpoints REST seguros y escalables con respuestas JSON consistentes.
- Diseño listo para integración con plataformas de marketing, CRMs y herramientas de análisis.

## Conceptos Centrales

Una Plataforma de Datos de Clientes (CDP) se construye sobre un conjunto de conceptos centrales que definen cómo se captura, organiza y activa la información del cliente. Entender estos conceptos es esencial para aprovechar al máximo la API.

### Perfiles

Un perfil representa una identidad única de cliente en la CDP. Consolida atributos como nombre, email, información de documentos y señales de comportamiento. Los perfiles son persistentes y sirven como la base de todas las interacciones con clientes.

### Transacciones

Una transacción captura un evento o interacción asociada con un cliente. Las transacciones pueden incluir compras, validaciones u otras actividades relevantes para tu negocio. Están marcadas con tiempo y vinculadas a perfiles, proporcionando una vista histórica del comportamiento del cliente y cambios de estado.

### Variables de Decisión

Las variables de decisión son atributos dinámicos almacenados dentro de un perfil que influyen en cómo un cliente es segmentado o comprometido.

### Etiquetas

Las etiquetas te permiten categorizar y etiquetar perfiles basados en criterios personalizados, pueden representar segmentos como "Cliente VIP", "En Riesgo", o "Nivel de Compromiso: Alto".

Las etiquetas facilitan el filtrado y agrupación de perfiles.

### Segmentos

Un segmento es una colección dinámica de perfiles que comparten atributos comunes, variables de decisión o etiquetas. Los segmentos están basados en reglas y se actualizan continuamente, haciéndolos esenciales para personalización, targeting y análisis. También pueden usarse para activar acciones o visualizar datos efectivamente.
